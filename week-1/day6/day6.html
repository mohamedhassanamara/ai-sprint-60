Section 01

K-Means Clustering

The Algorithm Mechanics: Initialization → Assignment → Update → Iterate

K-Means is the workhorse of clustering algorithms. It partitions $n$ data points into $K$ non-overlapping groups (clusters) by iteratively refining cluster center positions called centroids. The algorithm is elegantly simple: place $K$ centroids, assign each point to its nearest centroid, move each centroid to the mean of its assigned points, and repeat until convergence.

The four phases of K-Means operate as follows:

1. Initialization: Select $K$ initial centroid positions. The naive approach is random selection from the data points. The improved method, K-Means++, chooses initial centroids that are spread out: the first is random, subsequent centroids are chosen with probability proportional to $D(x)^2$, the squared distance to the nearest existing centroid. This dramatically reduces bad initializations.

2. Assignment Step: For each data point $x_i$, compute its Euclidean distance to all $K$ centroids and assign it to the cluster whose centroid is nearest:

Euclidean Distance
$$d(x_i, \mu_k) = \sqrt{\sum_{j=1}^{p} (x_{ij} - \mu_{kj})^2}$$

where $x_i$ is data point $i$, $\mu_k$ is centroid $k$, $p$ is number of features, and $j$ indexes each feature dimension.

3. Update Step: Recompute each centroid as the arithmetic mean of all points currently assigned to that cluster: $\mu_k^{(new)} = \frac{1}{|C_k|}\sum_{x_i \in C_k} x_i$.

4. Iteration: Repeat steps 2–3 until centroids stop moving (or move less than a tolerance $\epsilon$), or a maximum iteration count is reached. Convergence is guaranteed because WCSS decreases monotonically at each step.

import numpy as np

Section 02

K-Means Mathematics

Lloyd's Algorithm, WCSS (Inertia), and the Elbow Method

Lloyd's Algorithm is the formal name for the standard K-Means procedure. Formally, it minimizes the Within-Cluster Sum of Squares (WCSS), also called Inertia. This objective function measures total compactness: how close each point is to its own cluster's centroid.

WCSS (Inertia) — The K-Means Objective Function
$$J = \text{WCSS} = \sum_{k=1}^{K} \sum_{x_i \in C_k} \| x_i - \mu_k \|^2$$

where $K$ = number of clusters, $C_k$ = set of points in cluster $k$, $\mu_k$ = centroid of cluster $k$, and $\|.\|^2$ = squared Euclidean distance.

K-Means is an NP-hard optimization problem in general, but Lloyd's algorithm provides an efficient heuristic. Each iteration reduces $J$ monotonically—the assignment step finds the best cluster for each point given fixed centroids, and the update step finds the best centroid given fixed assignments. The algorithm converges to a local minimum, hence the importance of multiple random restarts (n_init).

The Elbow Method for selecting $K$: Plot WCSS against $K = 1, 2, 3, \ldots$ The curve always decreases (adding more clusters always reduces within-cluster variance), but there is typically a sharp "elbow" where additional clusters yield diminishing returns. The $K$ at the elbow is a reasonable choice.

plt.show()

Hierarchical Clustering

Agglomerative (Bottom-Up) Tree Building, Dendrograms, and Linkage Criteria

Agglomerative Hierarchical Clustering takes the opposite approach from K-Means: instead of pre-specifying $K$, it builds an entire tree of nested clusterings. Start with $n$ singleton clusters (each point is its own cluster), then iteratively merge the two "closest" clusters until everything is in one cluster. The result is a dendrogram—a tree diagram that records every merge and the distance at which it occurred.

The critical design choice is the linkage criterion—how you measure "distance" between two clusters $A$ and $B$:

Single Linkage (MIN): $d(A, B) = \min_{a \in A, b \in B} d(a, b)$. Distance between the two nearest members. Creates long, chained clusters. Vulnerable to the "chaining effect."

Complete Linkage (MAX): $d(A, B) = \max_{a \in A, b \in B} d(a, b)$. Distance between the two farthest members. Produces compact, roughly equal-sized clusters but is sensitive to outliers.

Ward's Method: Merges the two clusters that result in the smallest increase in total within-cluster variance (WCSS). Tends to produce similarly sized, spherical clusters—often the best default choice.

DBSCAN

Density-Based Spatial Clustering: ε-Radius, MinSamples, Core/Border/Noise

DBSCAN (Density-Based Spatial Clustering of Applications with Noise) discovers clusters of arbitrary shape based on density—regions of high point concentration separated by low-density voids. Unlike K-Means, it doesn't require specifying $K$ and can identify noise points (outliers) that belong to no cluster.

DBSCAN has two hyperparameters:

ε (epsilon): The radius of the neighborhood around each point.

MinPts (min_samples): The minimum number of points required within the ε-neighborhood for a point to be classified as a core point.

Every data point is classified into one of three categories:

Core Point: Has ≥ MinPts neighbors within its ε-ball (including itself). These form the dense hearts of clusters.

Border Point: Falls within ε of a core point but doesn't have MinPts neighbors itself. It's on the edge of a cluster.

Noise Point: Neither core nor border. An outlier, labeled as -1.

The algorithm: (1) find all core points, (2) form clusters by connecting core points that are within ε of each other (density-reachable), (3) assign border points to the nearest core point's cluster, (4) label everything else as noise.

Football Analogy

Imagine a GPS heatmap of every Premier League player's positional data over a season. K-Means would force every player into exactly K groups, even if some roam uniquely. DBSCAN is smarter: it finds clusters where players crowd together (the "parking the bus" cluster of deep-block CBs, the "pressing trap" cluster of high-line forwards). The truly unique players—the Messi who drifts everywhere, the goalkeeper who plays as sweeper in positions no one else occupies—get labeled as noise. That's not failure; it's DBSCAN correctly saying "this player doesn't fit any archetype," which is valuable scouting intelligence.

Python 
from sklearn.cluster import DBSCAN
from sklearn.preprocessing import StandardScaler

# CRITICAL: always scale before DBSCAN (distance-based!)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X_players)

# eps=0.5 means ε-neighborhood radius, min_samples=5
db = DBSCAN(eps=0.5, min_samples=5)
labels = db.fit_predict(X_scaled)

n_clusters = len(set(labels)) - (1 if -1 in labels else 0)
n_noise = list(labels).count(-1)
print(f"Clusters found: {n_clusters}, Noise points: {n_noise}")
💡 Key Insight
To tune ε, use a k-distance plot: compute the distance to the k-th nearest neighbor for each point (where k = MinPts), sort these distances, and look for an elbow. The ε at the elbow is a good starting point.

The Curse of Dimensionality & PCA

Why High Dimensions Break Everything, and the Philosophy of Variance Maximization

The Curse of Dimensionality describes a set of phenomena that arise in high-dimensional spaces which don't occur in lower dimensions. As the number of features $p$ grows:

1. Distance concentration: All pairwise distances converge—the nearest neighbor and farthest neighbor become approximately equidistant. This destroys K-Means and KNN algorithms that rely on meaningful distance contrasts.

2. Volume explosion: Data becomes sparse. To maintain statistical density, the amount of data needed grows exponentially with dimensions. A dataset of 1000 points feels "dense" in 2D but becomes hopelessly sparse in 500D.

3. Feature redundancy: Many real-world features are correlated. A footballer's "total passes" and "pass completion count" carry overlapping information.

Principal Component Analysis (PCA) is a linear dimensionality reduction technique that addresses this. Its philosophy: find new axes (principal components) along which the data has maximum variance. The first PC captures the most variance, the second captures the most remaining variance while being orthogonal to the first, and so on. You then keep only the top $d \ll p$ components, discarding the rest as noise.

Football Analogy

Your scouting database tracks 50 metrics per player: goals, xG, tackles, interceptions, aerial duels, progressive carries, through balls, pressures, etc. Good luck visualizing a 50D point cloud. PCA is like a brilliant chief scout who says: "Look, 80% of the variation across all players can be summarized on two axes: Attacking Output (a weighted combo of goals, xG, key passes) and Defensive Workrate (tackles, pressures, interceptions). Let's just plot every player on those two axes." You go from 50 numbers to 2—and suddenly you can literally see the gap in your squad for a ball-progressing defender.

⚠️ Critical Warning
PCA is a linear method. It finds the best linear projection. If the meaningful structure in your data lies along a curved manifold (e.g., players arranged in an arc in feature space), PCA will fail. That's where t-SNE and UMAP come in.
Section 06

PCA Mathematics

The Covariance Matrix, Eigendecomposition, Eigenvectors, and Eigenvalues

PCA's math is beautiful. Given a centered data matrix $X$ with $n$ samples and $p$ features (each column has mean 0 after standardization):

Step 1 — The Covariance Matrix
$$\Sigma = \frac{1}{n-1} X^T X$$

$\Sigma$ is a $p \times p$ symmetric matrix. Entry $\Sigma_{ij}$ measures the covariance between features $i$ and $j$. Diagonal entries are variances.

Step 2 — Eigendecomposition
$$\Sigma \, v_k = \lambda_k \, v_k$$

Solve for $p$ eigenvector-eigenvalue pairs $(v_k, \lambda_k)$. Each eigenvector $v_k$ (principal component direction) defines a new axis in feature space. Each eigenvalue $\lambda_k$ tells you the variance captured along that axis.

Step 3 — Variance Explained
$$\text{Explained Variance Ratio}_k = \frac{\lambda_k}{\sum_{j=1}^{p} \lambda_j}$$

If λ₁=4.2, λ₂=1.8, λ₃=0.3, ..., then PC1 explains 4.2/(4.2+1.8+0.3+...) of total variance.

Step 4 — Projection
$$X_{\text{reduced}} = X \cdot W_d$$

where $W_d = [v_1 | v_2 | \ldots | v_d]$ is the $p \times d$ matrix of the top $d$ eigenvectors. The result is your data in $d$ dimensions.

Football Analogy

The eigenvectors are like the "master axes of player variation." PC1 might correspond to an attacking-vs-defending axis (eigenvector heavily weighted on goals, xG, key passes positively and tackles negatively). PC2 might be a possession-vs-directness axis. The eigenvalues tell you how much spread exists along each axis: if λ₁ is 10x bigger than λ₂, almost all meaningful difference between players is captured by that single attacking-vs-defending dimension. The projection is literally remapping every player from their 50-stat scouting profile to a 2D coordinate on a tactical scatter plot.

t-SNE

Non-Linear Dimensionality Reduction via Probability Distributions and Local Neighborhood Preservation

t-Distributed Stochastic Neighbor Embedding (t-SNE) is a non-linear technique designed specifically for visualization of high-dimensional data in 2D or 3D. Unlike PCA, which preserves global variance through linear projections, t-SNE focuses on preserving local neighborhoods—points that are close in high-D should remain close in low-D.

How it works:

Step 1 — High-D Probabilities: For each pair of points $(x_i, x_j)$ in the original space, compute a probability $p_{ij}$ that represents how likely $x_j$ is to be picked as $x_i$'s neighbor, using a Gaussian kernel centered on $x_i$:

t-SNE — High-D Conditional Probability
$$p_{j|i} = \frac{\exp\left(-\|x_i - x_j\|^2 / 2\sigma_i^2\right)}{\sum_{k \neq i} \exp\left(-\|x_i - x_k\|^2 / 2\sigma_i^2\right)}$$

$\sigma_i$ is set per point such that the effective number of neighbors matches the user-specified perplexity.

Step 2 — Low-D Probabilities: In the embedding space, use a Student's t-distribution (with 1 degree of freedom) instead of a Gaussian:

t-SNE — Low-D Student-t Probability
$$q_{ij} = \frac{(1 + \|y_i - y_j\|^2)^{-1}}{\sum_{k \neq l} (1 + \|y_k - y_l\|^2)^{-1}}$$

The heavy tails of the t-distribution allow dissimilar points to be pushed far apart in the embedding, alleviating the "crowding problem."

Step 3 — Minimize KL Divergence: Optimize the positions $y_i$ using gradient descent to minimize the Kullback-Leibler divergence between $P$ and $Q$: $KL(P \| Q) = \sum_{i \neq j} p_{ij} \log \frac{p_{ij}}{q_{ij}}$.

Perplexity (typical range: 5–50) controls the effective number of local neighbors each point considers. Low perplexity = tight local focus, high perplexity = broader neighborhood. It's roughly interpretable as "the number of close neighbors each point has."

Football Analogy

Think of t-SNE as arranging player profile cards on a physical tactics board. You can't represent 50 stats in 2D perfectly, so t-SNE makes a promise: "Players who are similar in the full 50D will sit next to each other on this board. Players who are different will be pushed apart." The perplexity is like the scout's field of view—set it to 5 and you get very tight micro-clusters (all left-footed Argentine inside forwards huddle together), set it to 50 and you get broader groupings (all attackers loosely clump). The key limitation: distances between clusters are meaningless. If Cluster A is 3 inches from Cluster B on the board, that doesn't mean those player types are similar. Only within-cluster proximity is trustworthy.

⚠️ t-SNE Caveats
t-SNE is for visualization only, not as a preprocessing step for downstream ML. It's non-parametric (can't transform new data), stochastic (different runs give different layouts), and global distances are not preserved. Always run with multiple perplexity values and random seeds to validate structure.

UMAP

Uniform Manifold Approximation & Projection: Topology, Local vs. Global, n_neighbors & min_dist

UMAP (Uniform Manifold Approximation and Projection) is a newer non-linear dimensionality reduction technique that is generally faster than t-SNE, scales better, and—crucially—preserves more global structure while still capturing local neighborhoods.

UMAP is grounded in topological data analysis and Riemannian geometry. The intuition: the data lies on a manifold (a curved surface) embedded in high-D space. UMAP approximates this manifold by building a weighted k-neighbor graph, then optimizes a low-D layout that preserves the topological structure of that graph.

Key hyperparameters:

n_neighbors (default 15): Controls the local-vs-global balance. Small values (5–10) preserve fine micro-structure (tight clusters, many small groups). Large values (50–200) capture broader macro-structure (fewer, larger groupings, more global topology). This is analogous to t-SNE's perplexity but UMAP's global preservation is superior.

min_dist (default 0.1): Controls how tightly points are packed in the embedding. Small values (0.0–0.1) produce dense, clumpy clusters—great for identifying discrete groups. Large values (0.5–1.0) spread points out, preserving the continuous nature of the data—useful when you expect a spectrum rather than categories.

UMAP is like the ultimate scouting visualization wall. With n_neighbors=5, you zoom into hyper-specific niches: left-footed deep-lying playmakers who press from the front vs. right-footed ones who sit deep. Use UMAP when: you need speed on large datasets (100K+ points), want to preserve global structure, or need to transform new unseen data. Section 09

The Unsupervised Pipeline

Chaining Standardization → PCA → K-Means for End-to-End Unsupervised Workflows

In production, unsupervised learning steps are almost never used in isolation. They're chained into a pipeline—a sequence of transformations that flows data from raw features to final cluster assignments. Sklearn's Pipeline ensures clean, reproducible workflows with no data leakage.

Why this specific chain?

1. StandardScaler: K-Means and PCA are both variance-sensitive. If "market_value" ranges 0–200M and "age" ranges 16–40, the market value dimension will completely dominate distance calculations. Standardization (z-score) puts all features on equal footing.

2. PCA: Reduces dimensionality, removing noise and multicollinearity. K-Means works better in lower dimensions (curse of dimensionality). You choose components to retain ~95% variance.

3. K-Means: Clusters the reduced, clean data into $K$ groups.

Python 
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
from sklearn.cluster import KMeans
import pandas as pd

# Build the pipeline
unsupervised_pipeline = Pipeline([
    ('scaler',  StandardScaler()),
    ('pca',     PCA(n_components=0.95)),   # auto-select d for 95% variance
    ('kmeans',  KMeans(n_clusters=5, init='k-means++', 
                      n_init=10, random_state=42))
])

# Fit and predict in one line
cluster_labels = unsupervised_pipeline.fit_predict(X_players_raw)

# Inspect PCA component count
n_components = unsupervised_pipeline.named_steps['pca'].n_components_
print(f"PCA retained {n_components} components for 95% variance")

# Profile the clusters
df = pd.DataFrame(X_players_raw, columns=feature_names)
df['cluster'] = cluster_labels
cluster_profiles = df.groupby('cluster').mean()
print(cluster_profiles.round(2))

Using sklearn's Pipeline guarantees: (1) No data leakage—scaler/PCA fit only on training data during cross-validation. (2) Reproducibility—the entire workflow is a single serializable object. (3) Clean code—one .fit_predict() call replaces 6 separate steps.

The Human Interview Vault
5 rigorous questions — answered in the voice of a senior engineer in a live interview
K-Means just assigned all my goalkeepers to the same cluster as my centre-backs. What went wrong?
Honestly, the very first thing I check every single time is scaling. If you didn't StandardScale your features, then a stat like "passes attempted" ranging from 0 to 3000 will completely drown out "goals per 90" ranging from 0 to 1.5. K-Means uses Euclidean distance, and large-magnitude features dominate that calculation. Scale first, cluster second—it's almost always the fix. If scaling doesn't help, look at whether your features actually distinguish those positions. GKs and CBs might genuinely overlap on defensive metrics; you may need to engineer features like "saves per 90" or "penalty area entries" that separate them.
Why would you ever choose DBSCAN over K-Means, and what's the biggest pain point with it?
I'd reach for DBSCAN when I have no idea how many clusters exist, when I expect non-spherical cluster shapes, or when I know there are outliers I want flagged automatically—K-Means forces every point into a cluster, including noise. The real pain point is tuning epsilon. There's no clean "elbow" method like K-Means gives you. You end up doing a k-distance plot and eyeballing the knee, and it's sensitive—change ε by 0.1 and you can go from 3 clusters to 30. Also, DBSCAN struggles badly when clusters have very different densities, because one global ε can't serve both a tight cluster and a sparse one. HDBSCAN fixes that, and honestly, in production I usually just jump straight to HDBSCAN now.
Can you use PCA and t-SNE interchangeably? When would you pick one over the other?
No, they solve fundamentally different problems. PCA gives you a linear, deterministic, invertible projection—I use it as a preprocessing step before feeding data into classifiers or clustering. It actually reduces dimensionality in a way downstream models can consume. t-SNE is purely a visualization tool.
